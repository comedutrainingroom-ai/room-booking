Data Flow สรุป

  1. ผู้ใช้ เข้าเว็บ → ล็อกอินผ่าน Google (Firebase Auth)
  2. Client ส่ง Firebase ID Token ไป Server เพื่อ verify ตัวตน
  3. Server ตรวจ Token → สร้าง/อัปเดต User ใน MongoDB → ส่ง Role กลับ
  4. ผู้ใช้ทำรายการ (จอง, แจ้งปัญหา ฯลฯ) → REST API → MongoDB
  5. Socket.io แจ้งเตือน Admin แบบ real-time เมื่อมีการจองใหม่
  6. Cron Job ส่ง email เตือนก่อนประชุม 1 ชม.

3. Data Models (ฐานข้อมูล)

3.1 Room — ห้องประชุม

FieldTypeรายละเอียด
nameStringชื่อห้อง (unique, สูงสุด 50 ตัว)
capacityNumberจำนวนที่นั่ง
equipment[String]อุปกรณ์ที่มี เช่น โปรเจคเตอร์, ไมค์
images[String]path รูปภาพห้อง
descriptionStringคำอธิบาย (สูงสุด 500 ตัว)
isActiveBooleanเปิด/ปิดใช้งาน

3.2 Booking — การจอง

FieldTypeรายละเอียด
roomObjectId → Roomห้องที่จอง
topicStringหัวข้อการจอง
user.name/email/phoneStringข้อมูลผู้จอง
startTime / endTimeDateเวลาเริ่ม-สิ้นสุด
statusEnumpending · approved · rejected · cancelled
reminderSentBooleanส่ง email เตือนแล้วหรือยัง
isImportedBooleanจองจากการ import Excel

3.3 User — ผู้ใช้

FieldTypeรายละเอียด
emailStringอีเมล (unique, lowercase)
nameStringชื่อ
roleEnumstudent · admin
studentIdStringรหัสนักศึกษา
isBannedBooleanถูกแบนหรือไม่

3.4 Setting — ตั้งค่าระบบ (Singleton)

Fieldค่าเริ่มต้นรายละเอียด
systemNameระบบจองห้องประชุมออนไลน์ชื่อระบบ
maxBookingDays30จองล่วงหน้าได้กี่วัน
maxBookingHours4จองได้นานสุดกี่ชม./ครั้ง
requireApprovaltrueต้องรออนุมัติหรือไม่
openTime / closeTime08:00 - 20:00เวลาเปิด-ปิดจอง

3.5 Report — แจ้งปัญหา

FieldTypeรายละเอียด
topicStringหัวข้อปัญหา
urgencyEnumnormal · urgent · emergency
statusEnumpending · in_progress · resolved · rejected
roomObjectId → Roomห้องที่มีปัญหา
reporterObjectId → Userผู้แจ้ง

3.6 AuditLog — บันทึกกิจกรรม

FieldTypeรายละเอียด
actionEnumเช่น booking:create, user:ban
performedByObjectId → Userใครทำ
targetTypeEnumbooking, report, user, settings, room
detailsStringคำอธิบาย
📝 หมายเหตุAuditLog มี TTL index ลบข้อมูลอัตโนมัติหลัง 90 วัน

5. ระบบ Authentication & Authorization

5.1 Flow การล็อกอิน

sequenceDiagram participant U as ผู้ใช้ participant C as Client (React) participant FB as Firebase Auth participant S as Server (Express) participant DB as MongoDB U->>C: กดปุ่ม Login with Google C->>FB: signInWithPopup(googleProvider) FB-->>C: Firebase User + ID Token C->>S: POST /api/auth/google S->>FB: verifyIdToken(idToken) FB-->>S: Decoded token (email, name) S->>DB: findOneAndUpdate(User) DB-->>S: User doc (with role) S-->>C: success + user data C->>C: เก็บ dbUser ใน AuthContext

5.2 Middleware ป้องกัน API

// ทุก request ที่ต้อง login จะผ่าน protect middleware:
// 1. อ่าน Bearer Token จาก header
// 2. verify กับ Firebase Admin SDK
// 3. ค้นหา User จาก MongoDB
// 4. ใส่ req.user ให้ controller ใช้

// สำหรับ Admin routes จะเพิ่ม admin middleware:
// ตรวจ req.user.role === 'admin'

5.3 Admin Guard (2FA ด้วย PIN)

App.jsx
└── AdminGuard.jsx → ตรวจ isAdmin + isAdminUnlocked
    └── ถ้ายังไม่ unlock → แสดงหน้ากรอก PIN
    └── ถ้า unlock แล้ว → แสดงหน้า Admin ปกติ

7. ระบบ Real-time (Socket.io)

sequenceDiagram participant Admin as Admin Browser participant Server as Server participant Student as Student Browser Admin->>Server: socket.emit join-admin Note over Server: Admin อยู่ใน admin-room Student->>Server: POST /api/bookings สร้างจอง Server->>Server: บันทึกลง MongoDB Server->>Admin: io.to admin-room emit new-booking Note over Admin: แสดง notification / refresh ข้อมูล

8. ระบบ Email (Nodemailer)

Eventฟังก์ชันรายละเอียด
สร้างจองสำเร็จsendBookingCreated()แจ้งผู้จองว่ากำลังรออนุมัติ
จองได้รับอนุมัติsendBookingApproved()แจ้งผู้จองว่าอนุมัติแล้ว
เวลาจองถูกแก้ไขsendBookingModified()แจ้งเวลาเดิม/ใหม่
จองถูกยกเลิกsendBookingCancelled()แจ้งการยกเลิก
ใกล้ถึงเวลาประชุมsendBookingReminder()เตือน 1 ชม. ก่อน (Cron)
⚠️ สำคัญต้องตั้งค่า EMAIL_USER และ EMAIL_PASS ในไฟล์ server/.env — สำหรับ Gmail ต้องใช้ App Password

10. แนวทางการเพิ่มฟีเจอร์ใหม่

เพิ่ม Model ใหม่

1. สร้างไฟล์ใน server/models/NewModel.js
2. กำหนด Schema ด้วย Mongoose
3. ส่ง export: module.exports = mongoose.model('NewModel', schema)

เพิ่ม API Endpoint ใหม่

1. สร้าง Controller: server/controllers/newController.js
   → เขียน function: getAll, create, update, delete
   → ใช้ try-catch + ส่ง JSON { success, data/error }

2. สร้าง Route: server/routes/newRoutes.js
   → กำหนด protect / admin middleware

3. ลงทะเบียนใน server.js:
   app.use('/api/new', require('./routes/newRoutes'));

เพิ่มหน้าใหม่ใน Client

1. สร้างไฟล์: client/src/pages/NewPage.jsx
2. Lazy load ใน App.jsx:
   const NewPage = lazy(() => import('./pages/NewPage'));
3. เพิ่ม Route ใน App.jsx
4. เพิ่มลิงก์ใน Sidebar.jsx
